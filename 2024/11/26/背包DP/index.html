# 背包 DP

# 0-1 背包问题

# 问题

有nnn 个物品和一个容量为WWW 的背包，每个物品的重量为wiw_iwi​，价值为viv_ivi​．求物品的最大总价值，其中物品总重不超过背包容量．

0-1 背包问题是所有背包问题的鼻祖．

每个物品只有「在背包里」和「不在背包里」两种状态，对应二进制中的111 和000，因而得名．

# 思路

背包问题的经典解法是动态规划．「执古之道，以御今之有」便是动态规划的核心思想，其基本步骤如下：

  1. 明确问题（我在干啥） 用容量为WWW 的背包去装编号为1,2,⋯ ,n1, 2, \cdots, n1,2,⋯,n 的物品，求最大总价值．

  2. 定义状态（找出问题的参变量） 上面的问题我们记作Sol(n,W)\text{Sol}(n, W)Sol(n,W)．Sol\text{Sol}Sol 意为Solution\text{Solution}Solution，即解决方案；

  3. 拆解子问题（当前状态可以从哪些状态得来） 这里我们先以全局视角看待问题．我们关注编号为nnn 的物品：在整个问题的最优解里，这个物品在不在背包里？

    • 在背包里．我们把这个物品放进背包里，背包价值已达vnv_nvn​，那背包还剩多少空间呢？

      还剩W−wnW-w_nW−wn​．这个策略产生了一个子问题：

      「用容量为W−wnW-w_nW−wn​ 的背包去装编号为1,2,⋯ ,n−11, 2, \cdots, n-11,2,⋯,n−1 的物品，求最大总价值」，即Sol(n−1,W−wn)\text{Sol}(n-1, W-w_n)Sol(n−1,W−wn​)．

      整个问题的解决方案便产生了：Sol(n,W)=vn+Sol(n−1,W−wn)\text{Sol}(n, W) = v_n + \text{Sol}(n-1, W-w_n)Sol(n,W)=vn​+Sol(n−1,W−wn​)，wn≤Ww_n \le Wwn​≤W．

    • 不在背包里．编号为nnn 的物品被忽视，这个策略也产生了一个子问题：

      「用容量为WWW 的背包去装编号为1,2,⋯ ,n−11, 2, \cdots, n-11,2,⋯,n−1 的物品，求最大总价值」，即Sol(n−1,W)\text{Sol}(n-1, W)Sol(n−1,W)．

      整个问题的解决方案便产生了：Sol(n,W)=Sol(n−1,W)\text{Sol}(n, W) = \text{Sol}(n-1, W)Sol(n,W)=Sol(n−1,W)．

  4. 状态转移（参考已解决子问题的答案，提出当前未解决问题的方案） 两种策略分析完毕后，为了解决整个问题，我们该采取哪种策略呢？由于「在背包里」和「不在背包里」这两种情况是互补的，已经涵盖整个问题，故两者取最优即可 （最优子结构），即

    Sol(n,W)=max⁡(vn+Sol(n−1,W−wn), Sol(n−1,W))\text{Sol}(n, W) = \max(v_n + \text{Sol}(n-1, W-w_n),\, \text{Sol}(n-1, W)) Sol(n,W)=max(vn​+Sol(n−1,W−wn​),Sol(n−1,W))

    注意，我们策略的选择可能并不自由，因为某些情况下我们没得选：在这里当前物品可能太重装不下，只能采取「不在背包里」这个策略：

    Sol(n,W)=Sol(n−1,W)\text{Sol}(n, W) = \text{Sol}(n-1, W) Sol(n,W)=Sol(n−1,W)

    我们发现接下来的问题和原来的问题形式上是一样的，只是问题的规模变小了．到这里我们有很自然的实现方式：递归．任意小规模问题的方案与上式形式一样，我们考虑小规模问题「用容量为ccc 的背包去装编号为1,2,⋯ ,i1, 2, \cdots, i1,2,⋯,i 的物品」，则

    Sol(i,c)=max⁡(vn+Sol(i−1,c−wn), Sol(i−1,c))\text{Sol}(i, c) = \max(v_n + \text{Sol}(i-1, c-w_n),\, \text{Sol}(i-1, c)) Sol(i,c)=max(vn​+Sol(i−1,c−wn​),Sol(i−1,c))

    我们发现在递归过程中，问题的规模始终是单调减小的（无后效性），因此我们在关注物品1,2,⋯ ,i1, 2, \cdots, i1,2,⋯,i 的时候，「对物品iii 是否放进背包中的决策」不会受「背包中物品i+1,i+2,⋯ ,ni+1, i+2, \cdots, ni+1,i+2,⋯,n 的存在与否」的影响．

    在递归过程中，一个问题可能同时是多个大问题的子问题（子问题重叠），因此我们需要用数组dp[i][c]\text{dp}[i][c]dp[i][c] 存储子问题的答案以供后续大问题的查阅，并干脆直接使用递推实现状态的转移：

    dp[i][c]=max⁡(v[i]+dp[i−1][c−w[i]], dp[i−1][c])\text{dp}[i][c] = \max(v[i] + \text{dp}[i-1][c-w[i]],\, \text{dp}[i-1][c]) dp[i][c]=max(v[i]+dp[i−1][c−w[i]],dp[i−1][c])

  5. 解决基本小问题 由于问题规模始终单调减小，递归终将产生基本小问题．在这里，基本小问题应该是Sol(1,0)\text{Sol}(1, 0)Sol(1,0)，Sol(1,1)\text{Sol}(1, 1)Sol(1,1)，⋯\cdots⋯，Sol(1,W)\text{Sol}(1, W)Sol(1,W) 了吧，不过我们还可以更进一步：Sol(0,0)\text{Sol}(0, 0)Sol(0,0)，Sol(0,1)\text{Sol}(0, 1)Sol(0,1)，⋯\cdots⋯，Sol(0,W)\text{Sol}(0, W)Sol(0,W)，这些问题的答案显然都是000．

至此，我们只需要从头到尾遍历一遍dp\text{dp}dp 数组便可以得到整个问题的答案：dp[n][W]\text{dp}[n][W]dp[n][W]．

# 代码

时间复杂度：O(nW)O(nW)O(nW)，空间复杂度：O(nW)O(nW)O(nW)．

# 优化

O(nW)O(nW)O(nW) 的空间复杂度并不优秀．我们使用了二维数组dp\text{dp}dp 存储状态，但是我们会发现在状态转移到iii 的过程中，只使用到了i−1i-1i−1 的状态，i−2,i−3,⋯ ,2,1i-2, i-3, \cdots, 2,1i−2,i−3,⋯,2,1 的状态对当前及以后的决策毫无帮助，无疑浪费了存储空间．无独有偶，w[i]w[i]w[i] 和v[i]v[i]v[i] 自始至终只被使用过一次用于状态转移．

很自然的想法便是边读w,vw, vw,v 边转移dp\text{dp}dp，并只存储两层的状态dp[2][W]\text{dp}[2][W]dp[2][W]，每层转移新老交替，循环利用．但我们可以更进一步：只使用一层存储状态dp[W]\text{dp}[W]dp[W]，新老状态共用，这便是滚动数组：

dp[c]=max⁡(v+dp[c−w], dp[c])\text{dp}[c] = \max(v + \text{dp}[c-w],\, \text{dp}[c]) dp[c]=max(v+dp[c−w],dp[c])

如果沿用之前的思路（背包容量ccc 从000 到WWW 遍历），我们很快就会发现问题：dp[c−w]\text{dp}[c-w]dp[c−w] 已经不代表dp[i−1][c−w[i]]\text{dp}[i-1][c-w[i]]dp[i−1][c−w[i]] 了，而是早已被更新为dp[i][c−w[i]]\text{dp}[i][c-w[i]]dp[i][c−w[i]]，因为状态的转移是按照背包容量从小到大的顺序进行的．此时物品iii 已经被考虑了，而不仅仅考虑物品1,2,⋯ ,i−11, 2, \cdots, i-11,2,⋯,i−1，这就导致后面的大背包容量问题没法使用小规模问题「仅考虑物品1,2,⋯ ,i−11, 2, \cdots, i-11,2,⋯,i−1」的答案dp[i−1][⋯ ]\text{dp}[i-1][\cdots]dp[i−1][⋯]．该怎么办？

很简单，只需将遍历顺序倒转过来即可（背包容量ccc 从WWW 到000 遍历）：这保证了大背包容量问题能够使用小规模问题「仅考虑物品1,2,⋯ ,i−11, 2, \cdots, i-11,2,⋯,i−1」的答案并优先得到解答．

# 优化代码

时间复杂度：O(nW)O(nW)O(nW)，空间复杂度：O(W)O(W)O(W)．
